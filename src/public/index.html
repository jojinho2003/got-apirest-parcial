API RESTful — Game of Thrones

Proyecto: Parcial 1 - Aplicaciones Híbridas

Descripción

Esta API expone información de Characters (personajes) y Houses (casas) del universo Game of Thrones. Implementada en Node.js / Express y MongoDB (Mongoose).

Endpoints principales

Characters
  • GET /api/characters — Ver todos los personajes
  • GET /api/characters/:id — Obtener personaje por ID
  • POST /api/characters — Crear personaje (body JSON)
  • PUT /api/characters/:id — Actualizar personaje
  • DELETE /api/characters/:id — Eliminar personaje
Houses
  • GET /api/houses — Ver todas las casas
  • GET /api/houses/:id — Obtener casa por ID (incluye miembros)
  • POST /api/houses — Crear casa
  • PUT /api/houses/:id — Actualizar casa
  • DELETE /api/houses/:id — Eliminar casa (desvincula personajes)

Validaciones (breve)

Al crear/actualizar personajes se valida que name exista, que age sea entero >= 0 y que alive sea booleano. Para casas se valida name requerido y único.

Cómo usar (resumen)

  1. Levantar MongoDB local o Atlas.
  2. Configurar `.env` con `MONGO_URI` y `PORT`.
  3. Instalar dependencias `npm install` y correr `npm run dev`.
  4. Opcional: correr `npm run seed` para poblar algunos datos de ejemplo.
